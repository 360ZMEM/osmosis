## 3. 这些结果意味着什么

### 3.1 时间契约 ✓
- 1400 step ≈ 23 s 仿真 ≈ 12–17 s wall-clock（CPU 仿真比 1:2 实时）。
- 4 组 rc=0，CSV 1401 行完整，`buffer.pt + summary.json + 6 PNG` 全部产出。

### 3.2 STDW 在漂移期"压顶"效应明确
- baseline 漂移期 `max=18.71`，stdw_only/filter 降到 `9.71`（**−48%**）。
- stdw_only/filter 的 `mean_drift = 0.965` 也低于 baseline 的 `1.382`（**−30%**）。
- 这说明 STDW + Lyapunov mask + frozen-ref 锚点在 yaw 通道漂移最严重的中段，**已经能成功抑制极端尖峰**。

### 3.3 但 post-drift 反而恶化
- baseline post `mean=0.888 max=3.27`；stdw_* post `mean=1.78~2.35 max=11.75~12.40`。
- 1400 步内 STDW 微调把 policy 推离原 SS4 平衡点，但远没收敛到"新分布稳态"，处于**两边都不优的过渡期**。
- 表现为：`convergence_step` baseline=289（恢复到 SS4 收敛）/ stdw_filter=None / stdw_full=None。

### 3.4 stdw_full 反而最差
- `mean=1.493 max=18.02`，几乎和 baseline 持平。
- 原因：`use_quantile_filter` 默认 `discard_ratio=0.1`，剔除 buffer 中 error 最大的 10%。
- STDW **真正需要纠正的恰恰是这些尖峰样本**，被 quantile 扔掉后，target 信号只剩"已经稳定"的样本，等同退化到 baseline + 噪声。

### 3.5 stdw_only ≡ stdw_filter
- 两组 `mean/max/L_src/L_tgt/loss` 全部一致。
- 因为 `enable_filter` 只影响 buffer `error` 字段，不参与 loss 计算（`error` 仅给 quantile_filter 用）。
- 这是当前设计的**逻辑后果**而非 bug。

## 4. 为什么会出现这些现象

### 4.1 STDW 已生效但更新量级太小
- 修复后实测：每次慢环 `loss ≈ 1.5e-2`，乘以 lr=5e-5 → `Δθ ~ 7.5e-7` 单步。
- 19 次累积 `Δθ ~ 1.4e-5`（与 `L_reg ~ 6e-4 = ‖Δθ‖²` 反推一致）。
- SS4 训练时 actor 参数典型量级 ~10⁻¹，相对扰动 10⁻⁴ 量级，**只够"轻推"少数极端样本，不够整体重塑**。
- 这正解释了 §3.2（漂移期尖峰被压）+ §3.3（post-drift 整体没收敛到新最优）。

### 4.2 pseudo_action 信号弱
- `pseudo_action = action + 1.0 * J_inv_diag * Δu`，其中 `Δu = _pid_value_add_buf` 是底层 PID 自适应增量。
- buffer 实测 `|pseudo - action|.mean = 0.024`，`mse_tgt ~ 0.024² ≈ 6e-4`，mask 后 ~3e-2（与 §2.4 一致）。
- 想让 STDW 更激进，要么**加大 `pseudo_gain`**（直接放大目标差），要么**改 PID 使 Δu 本身更大**。

### 4.3 quantile filter 剔除了关键信息
- `discard_ratio=0.1` 让 stdw_full 损失了 10% 高 error 样本。
- 这些样本恰是 yaw 漂移最严重的时刻 (`max~18`)。
- 结果：stdw_full `drift_max=18.02` 比 stdw_only `9.71` 高 86%，几乎退化回 baseline。

### 4.4 训练步数太少
- 1400 步 / 慢环间隔 60 → 只有 **19 次更新**。
- 对于 actor MLP（9→64→64→4，约 5K 参数），19 个 minibatch 远不够把 PPO 从 SS4 收敛点搬运到新分布。

## 5. 已知遗留行为（非 bug）

| 现象 | 原因 |
|---|---|
| stdw_only / stdw_filter 完全同分 | `enable_filter` 只影响 `error` 字段，不入 loss（设计如此）|
| `L_src` 量级远小于 `L_tgt` | 19 次更新太少，policy 还没明显偏离 frozen ref |
| post-drift `final_mse` 反升 | 1400 步无法在 STDW 下完整收敛，处于过渡期 |
| `L_src.min == 0` 偶发 | 全 mask 屏蔽 + 0/clamp_min → 退化为 0，实际 effective_batch_frac=0 → 该次不 step |

### 8.1 修复 7、8 改动概览

- **修复 7**（[L147-154](file:///home/zmem063/isaaclab/source/extensions/omni.isaac.lab_tasks/omni/isaac/lab_tasks/direct/isaac-auv-env-new/workflows_new_stdw/play_stdw_adapt.py#L147-L154)）：CLI 暴露 `--g_C_lr / --pseudo_gain / --slow_loop_interval / --lambda_reg / --reg_mode`，新参 `reg_mode ∈ {l2, behavior_kl}` 默认 `behavior_kl`。

### 8.5 这意味着什么

- **早期 ✅**：drift_early `mean -35%` 表明高 lr + 大 pseudo_gain + 关闭 quantile + behavior_kl 这一套**确实能在漂移开始时迅速压低误差**，方向正确。

### 8.6 为什么早期赢、后期输

1. **累积 Δθ 已达 SS4 训练量级 5–10%**：`g_C_lr=1e-3 × 287 步 × loss~3.87e-3 ≈ 10⁻³`，对单层 `nn.Linear(64,4)` 而言已足以让某些 yaw 输出权重 sign 翻转。
2. **Behavior-KL 仅锚 source 分布**：`L_reg = MSE(π(s_src), π_ref(s_src))` 只在静水观测上拉回；在漂移观测 `s_tgt` 上，policy 完全自由漂移。drift_late 段 `s_tgt` 占主导，约束失效。
3. **pseudo_action 在长漂移下也会失稳**：底层 PID `Δu` 累积积分项随时间放大，`pseudo_gain=5.0` 把它进一步放大 → 慢环目标本身已漂移，policy 跟着追错的目标。
4. **没有早停**：drift_late + post 一直在做无效更新，越拉越差。

## §9. 双向 KL + 伪标签门控/衰减（改良 1+2）与 sweep8

§8 暴露的核心问题：tuned 6000-step 在漂移末段反而比 baseline 劣化（mean +23%、drift_late +146%、post +50%）。§8.7 提出 α-ζ 修正方向，本节落地其中三个：

- **α** 把 drift_end 从 1200 推到 ≈ total_steps × 0.85，让 ρ 缓慢爬升，避免 ρ 阶跃；
- **β** 把 g_C_lr 从 5e-5 拉到 1e-3、slow_loop_interval 从 60 缩到 20；
- **γ** 引入双向行为锚定（改良 1）；
- **δ** 引入伪标签 clip-gate 与 ρ-decay（改良 2）；
- **ε** 修复 self_adapt 默认关闭导致 pseudo-action 链路完全失效的"沉默 bug"。

### §9.1 改良 1：双向行为锚定（Bi-directional Behavioral KL）

旧版 behavior_kl 只在 source obs 上锚 ref，目标域观测可"放飞"。改为：

$$\mathcal{L}_{reg} = (1-\rho)\,\text{MSE}\big(\pi_\theta(s_{src}),\,\pi_{ref}(s_{src})\big) + \rho\,\text{MSE}\big(\pi_\theta(s_{tgt}),\,\pi_{ref}(s_{tgt})\big)$$

实现：[play_stdw_adapt.py#L688-722](file:///home/zmem063/isaaclab/source/extensions/omni.isaac.lab_tasks/omni/isaac/lab_tasks/direct/isaac-auv-env-new/workflows_new_stdw/play_stdw_adapt.py#L688-L722)（同时算 `a_src_anchor`、`a_tgt_anchor`，按 ρ 加权）。

### §9.2 改良 2：伪标签 clip-gate + ρ-decay

旧版 `a_pseudo = clip(action + pseudo_gain·J⁻¹·Δu, -1, 1)` 没有上界，PID 积分饱和会反哺出过激动作；并且 ρ→1 时低层修正应让位给策略。改为：

1. **Clip-gate**：`correction = clip(scaled_gain · J⁻¹ · Δu, -gate, +gate)`（默认 gate=0.5）
2. **ρ-decay**：`scaled_gain = pseudo_gain · max(0, 1 - pseudo_decay·ρ)`（默认 decay=0.7）

实现：[play_stdw_adapt.py#L607-628](file:///home/zmem063/isaaclab/source/extensions/omni.isaac.lab_tasks/omni/isaac/lab_tasks/direct/isaac-auv-env-new/workflows_new_stdw/play_stdw_adapt.py#L607-L628)。

### §9.3 修复 ε：A-S-Surface 必须显式启用

[warpauv_env.py#L136](file:///home/zmem063/isaaclab/source/extensions/omni.isaac.lab_tasks/omni/isaac/lab_tasks/direct/isaac-auv-env-new/warpauv_env.py#L136) 默认 `self_adapt = False`；STDW workflow 从未调用 `apply_control_profile(...)` →
