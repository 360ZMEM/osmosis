# 控制器稳定调节记录（参考轨迹 + 阻尼奖励）

> 目的：把跟踪曲线（`stdw_tracking_overlay.png`）从「阶跃方波 + 欠阻尼超调」调成「平滑正弦 + 可读波形」的全过程、参数、调节方法固化下来，便于复现与续调。
>
> 适用任务：`EasyUUV-Direct-Parametric-v1`（8 维 meta-control，`experiment_name=easyuuv_parametric`）。
> 最近一次有效配置：2026-06-07 `logs/rsl_rl/easyuuv_parametric/2026-06-07_16-27-58_stage1/model_1199.pt`。

---

## 1. 问题与诊断

初版 overlay 图存在两类「不美观」：

1. **超调 / 欠阻尼振荡**：参考目标是「每个 episode 一个随机姿态硬阶跃」（`_reset_goal` 调 `random_orientation`），阶跃后姿态欠阻尼振荡。
2. **曲线像斜坡而非波形**：改成正弦后，初版正弦 **幅度太小（0.15 rad）** 且 **周期远长于 episode**（freq 0.10 Hz → 周期 10s，而 episode 仅 3s），单个 episode 内只能看到 ~1/4 个波形，视觉上像斜坡。

> 关键验证（避免误判 bug）：从 play 的 `stdw_output.csv` 读 `des_roll/des_pitch/des_yaw` 逐步数值，确认正弦**确实在持续注入**（如 des_pitch 在相邻步从 -0.0727 平滑变到 -0.0978，覆盖 ±幅度满量程）。`episode_reset` 仅在 episode 边界为 1。所以「稳定后看似平直」是 **Actual 被钳在 0 / 误差恒定**，不是参考没动。

诊断结论：
- 加大阻尼奖励权重（rate/jerk/ang_vel）对高频纹波**收益递减**，单纯堆权重 MSE 几乎不动。
- 真正见效的是：**延长 episode + 放大正弦幅度 + 提高正弦频率**，让单个 episode 内出现完整波形。
- 副产物：episode 从 3s→6s 后，**深度（Depth）跟踪也大幅改善**（MSE 1.14→0.06），说明之前深度「下沉」主要是 3s 太短没收敛完，而非控制 bug。

## 2. 代码改动位置（easyuuv_env.py）

所有改动向后兼容：基类 `EasyUUVEnvCfg` 默认 `reference_mode="step"` + 阻尼权重为 0，保持旧 4 维基线行为不变；只有 `EasyUUVParametricEnvCfg` 与 CLI 显式启用新行为。

### 2.1 配置项（基类 `EasyUUVEnvCfg`）

```python
# === 参考轨迹（reference）生成 ===
reference_mode = "step"          # "step" 硬阶跃；"sine_sweep" 平滑正弦
ref_sine_amp  = [0.3, 0.3, 0.3]  # 逐轴幅度 (rad) [roll,pitch,yaw]
ref_sine_freq = [0.10, 0.13, 0.07]  # 逐轴频率 (Hz)

# === 阻尼奖励权重 ===
rew_scale_ang_vel     = 0.0  # 角速度阻尼
rew_scale_action_rate = 0.0  # 相邻动作差 ‖a_t - a_{t-1}‖，抑制抖动
rew_scale_action_jerk = 0.0  # 动作二阶差（jerk），压高频震荡
```

### 2.2 当前生效配置（`EasyUUVParametricEnvCfg`，约 line 1410）

```python
@configclass
class EasyUUVParametricEnvCfg(EasyUUVEnvCfg):
    num_actions = 8
    tune_gains = True
    episode_length_s = 6.0              # ← 3.0 改 6.0：单 episode 走完约 1 个正弦周期
    reference_mode = "sine_sweep"
    ref_sine_amp  = [0.35, 0.35, 0.35]  # ← ~20°，波形明显
    ref_sine_freq = [0.18, 0.22, 0.15]  # ← 提高频率，episode 内可见完整波形
    rew_scale_ang_vel     = 0.05
    rew_scale_action_rate = 0.02
    rew_scale_action_jerk = 0.01
```

### 2.3 实现要点

| 位置 | 作用 |
|---|---|
| `__init__`（约 276） | 新增 `_ref_phase(N,3)`、`_prev_action`、`_prev_prev_action` 缓冲 |
| `_pre_physics_step`（约 724） | 覆盖动作前存 prev/prev_prev；sine_sweep 模式每控制步调 `_update_reference()` |
| `_reset_idx`（约 881） | 重置 prev 动作缓冲 |

## 3. 调节方法（怎么调）

### 3.1 让波形「更明显 / 更平滑」

| 想要的效果 | 调哪个 | 方向 |
|---|---|---|
| 波形幅度更大（动作更明显） | `ref_sine_amp` | ↑（如 0.15→0.35），注意别超过物理可达姿态 |
| 单个 episode 内看到更多个完整周期 | `ref_sine_freq` 或 `episode_length_s` | freq↑ 或 episode↑（周期数 ≈ freq × episode_length_s） |
| 曲线整体更平滑、瞬态占比更低 | `episode_length_s` | ↑（3→6 已显著改善深度与瞬态） |
| 抑制叠加在正弦上的高频抖动 | `rew_scale_action_rate` / `rew_scale_action_jerk` | ↑（但收益递减，见下） |
| 抑制角速度方向的振荡 | `rew_scale_ang_vel` | ↑ |

> 周期数估算：`周期数 ≈ ref_sine_freq[axis] × episode_length_s`。
> 当前 Roll：0.18 × 6 ≈ 1.08 个周期/episode。

### 3.2 阻尼权重的经验

- rate/jerk/ang_vel 从极低幅度（0.01~0.02）起步，**翻 2–3 倍**对高频纹波改善有限（实测两轮 MSE 基本持平）。
- 阻尼项是高斯型 `scale·exp(-‖diff‖²)`：diff 大时趋近 0，diff 小时趋近 scale，本质是「平滑加分」而非「振荡重罚」。若要强压振荡，需改成线性/二次惩罚（负号项），属下一步专项。

### 3.3 CLI 课程式覆盖（不改代码临时试参）

`workflows/train_meta.py` 已支持以下透传（默认 None=沿用 cfg）：

```bash
--reference_mode {step,sine_sweep}
--ref_sine_amp  0.35,0.35,0.35     # 逗号分隔 roll,pitch,yaw
--ref_sine_freq 0.18,0.22,0.15
--rew_scale_ang_vel     0.05
--rew_scale_action_rate 0.02
--rew_scale_action_jerk 0.01
```

> 注意：`episode_length_s` 目前**没有** CLI，需直接改 `EasyUUVParametricEnvCfg`。

## 6. Roll/Yaw 纹波根因诊断（隔离实验）

对 `model_1199.pt` 做了三组零/低成本隔离实验，逐一排除嫌疑：

| 实验 | 做法 | Roll/Pitch/Yaw MSE | 结论 |
|---|---|---|---|
| 基线 | use_stdw=True, noise=0.02 | 9.91e-03 / 4.32e-03 / 9.38e-03 | 纹波明显 |
| C2 关 STDW | `--use_stdw False` | 8.17e-03 / 5.52e-03 / 7.56e-03 | 纹波**几乎不变** → STDW 非主因（反而对 Depth 稳定有正贡献） |
| C1 关噪声 | `--noise_std 0.0` | 6.63e-03 / 2.33e-03 / 7.99e-03 | 纹波形态/频率**几乎不变** → 观测噪声非主因 |

**纹波频率量化**（关噪声数据，误差去趋势后过零计数）：

| 轴 | 纹波幅度 std | 主频 | nominal ζ1(P) |
|---|---|---|---|
| Roll  | 0.068 | 4.96 Hz | 0.6/0.6=1.0 |
| Pitch | 0.049 | 2.74 Hz | 0.6/0.6=1.0 |
| Yaw   | 0.093 | 3.57 Hz | 1.0/0.6≈1.67 |

**结论（根因已锁定）**：
1. 纹波是 **2–5 Hz 低频**，远低于控制频率(60Hz)/物理(120Hz)，**排除 decimation 子步跳变 bug**（那会是 30–60Hz）。
2. 纹波幅度排序（Yaw>Roll>Pitch）与 **P 增益 ζ1 排序一致**（Yaw 最高 1.67x）→ 典型**闭环欠阻尼极限环**：P 偏高 + D 阻尼不足导致持续自激振荡。
3. 根因落在 **控制器增益与阻尼**，与参考轨迹/STDW/噪声无关。

**下一步专项候选**（按预期性价比排序）：
- B1（最快验证）：降 Yaw 比例增益 ζ1（1.67→0.6~0.8）或抬 Yaw 微分 ζ2，见 [easyuuv_env.py:155-158](../../easyuuv_env.py)。
- B2：降 `s_ratio`（4→2）软化 S 面，全局减抖。
- A1：S 面微分项 `a_d = actions - old_actions` 加低通滤波（EMA），抑制策略动作差分放大。
- A3：D 项从"动作差分"改为真实角速度 `-ζ2·ω_body`（更正统的速度阻尼），同时把角速度喂回观测（[L802-803](../../easyuuv_env.py) 当前被注释）。
- D1：阻尼奖励改为负向二次惩罚 `-scale·‖Δa‖²` 或 `-scale·‖ω‖²`（当前 `exp(-‖·‖²)` 是"平滑加分"，振荡大时趋 0 无惩罚——这正是上轮加权重无效的原因）。

> 复现隔离实验：`--use_stdw False` / `--noise_std 0.0` 即可，无需改代码、无需重训。

## 7. 修复尝试与结论

### 7.1 B1 + B2 同时（已验证 → 部分回退）

实施：`easyuuv_env.py` Yaw `ζ1=1.0→0.7、ζ2=0.13→0.20`（[L155-158](../../easyuuv_env.py)）+ `s_ratio=4→2`（[L162](../../easyuuv_env.py)）。重训 1200 iter（`logs/rsl_rl/easyuuv_parametric/2026-06-07_17-47-52_stage1/model_1199.pt`）。

| 轴 | 基线 MSE | B1+B2 MSE | ripple_std | fft_freq |
|---|---|---|---|---|
| Roll  | 9.91e-03 | 5.23e-03 ↓ | 0.062 ↓ | 3.83 Hz ↓ |
| Pitch | 4.32e-03 | 1.21e-02 ↑ | 0.093 ↑ | 2.00 Hz ↓ |
| Yaw   | 9.38e-03 | 6.91e-03 ↓ | 0.079 ↓ | 2.35 Hz ↓ |
| Depth | 6.13e-02 | 8.97e-01 **崩溃** | — | — |

结论：
- **B1（降 Yaw 增益）方向正确**：Yaw 纹波幅度/频率均下降。
- **B2（s_ratio 4→2）副作用大**：作为全局陡度参数一刀切，削弱了 Depth 推力（Actual 沉到 0.5），也让 Pitch 起始段超调到 0.6。
- 决策：**回退 B2，保留 B1，跳过剩余 B 尝试，进入 D1**。

### 7.2 D1 阻尼奖励改负向二次惩罚（已验证 → 喜忧参半）

实施：`easyuuv_env.py` jit `_compute_rewards`（[L1389-L1398](../../easyuuv_env.py)）三项从 `+scale·exp(-‖·‖²)` 改为 `-scale·‖·‖²`：
- `rew_ang_vel`、`rew_action_rate`、`rew_action_jerk` 全部变成真负向二次惩罚；scale 维持 0.05/0.02/0.01 不变。

5-iter dryrun 量级验证：vloss 44-54（基线 ~3-12），略放大但**未失控**，可进入 1200iter 正式训练。

正式训练：1200 iter 用时 ~365s，最终 reward 121.85、log MSE 2.59，与历史基线同量级（`logs/rsl_rl/easyuuv_parametric/2026-06-08_11-21-27_stage1/model_1199.pt`）。

play 评估对比（含 STDW + 噪声）：

| 轴 | 基线 (06-07 17:24) | B1+B2 (06-07 17:47) | **B1+D1 (06-08 11:30)** |
|---|---|---|---|
| Roll MSE / ripple_std / fft | 5.38e-02 / 0.057 / 2.43Hz | 5.23e-03 / 0.062 / 3.83Hz | **1.41e-02 / 0.042 / 1.91Hz** |
| Pitch MSE / ripple_std / fft | 5.46e-02 / 0.040 / 2.17Hz | 1.21e-02 / 0.093 / 2.00Hz | **1.62e-02 / 0.044 / 1.91Hz** |
| Yaw MSE / ripple_std / fft | 5.39e-02 / 0.030 / 2.17Hz | 6.91e-03 / 0.079 / 2.35Hz | **3.84e-03 / 0.039 / 1.74Hz** |
| Depth MSE | 6.13e-02 | 0.897 崩溃 | 1.13 仍崩溃 |

结论：
- **D1 直接达成主目标**：相对 B1+B2，三轴 `ripple_std` 全部下降 30-53%，主频从 2-4Hz 降到 1.7-1.9Hz。这是负向二次惩罚提供持续梯度的预期效应。
- **副作用：Roll/Pitch 跟踪 MSE 退步**：roll +170%、pitch +34%。负向项压住了高频抖动同时也压住了瞬时控制活力，bias 上升。Yaw 反而 MSE 改善（-44%），可能 yaw 纹波最重所以收益最大。
- **Depth 仍崩溃**：B1+D1 vs B1+B2 都是 ~0.9-1.1，远高于基线 0.06。说明 Depth 崩溃**不是 B2 引起**（已回退），而是 stage1 1200iter 在 episode_length=6s 配置下 Depth 不稳收敛——单训练种子方差。需要单独定位（或多 seed 平均）。
- 决策：D1 已验证为有效的纹波抑制手段；当前 scale 量级偏强，把跟踪 MSE 拖后腿。下一步进 A3（D 项改真实角速度）前，可考虑 (a) 减弱 D1 scale 一半（0.025/0.01/0.005）保留纹波抑制并恢复 MSE，或 (b) 直接进 A3 看其能否在不退跟踪的前提下进一步抑制纹波。Depth 崩溃问题需另立专项。

### 7.3 回退 B1 + D1 scale 缩减为 1/4（已验证 → 平衡点找到）

实施：
- **B1 回退**：`easyuuv_env.py` [L157](../../easyuuv_env.py#L157) Yaw 行 P/D 还原为 `1.0/PWM, 0.13/PWM`（撤销前次 0.7/0.20 增益调整）。
- **D1 缩减**：`EasyUUVParametricEnvCfg` [L1421-L1423](../../easyuuv_env.py#L1421-L1423) 三个 scale 全部降到原来的 1/4：
  - `rew_scale_ang_vel`：0.05 → **0.0125**
  - `rew_scale_action_rate`：0.02 → **0.005**
  - `rew_scale_action_jerk`：0.01 → **0.0025**

正式训练：1200 iter 用时 ~362s，vloss 收敛到 ~0.21（D1 full 时 ~50，量级回归正常），最终 reward 153、log MSE ~0.93，明显优于 D1 full 的 122。Checkpoint：`logs/rsl_rl/easyuuv_parametric/2026-06-08_11-37-26_stage1/model_1199.pt`。

play 评估对比（含 STDW + 噪声 + drift）：

| 轴 | 基线 (06-07 17:24) | B1+B2 (06-07 17:47) | B1+D1 (06-08 11:30) | **B1回退+D1×1/4 (06-08 11:44)** |
|---|---|---|---|---|
| Roll MSE / ripple_std / fft | 5.38e-02 / 0.057 / 2.43Hz | 5.23e-03 / 0.062 / 3.83Hz | 1.41e-02 / 0.042 / 1.91Hz | **3.95e-03 / 0.060 / 3.04Hz** |
| Pitch MSE / ripple_std / fft | 5.46e-02 / 0.040 / 2.17Hz | 1.21e-02 / 0.093 / 2.00Hz | 1.62e-02 / 0.044 / 1.91Hz | **3.65e-03 / 0.052 / 2.43Hz** |
| Yaw MSE / ripple_std / fft | 5.39e-02 / 0.030 / 2.17Hz | 6.91e-03 / 0.079 / 2.35Hz | 3.84e-03 / 0.039 / 1.74Hz | **2.16e-03 / 0.040 / 2.35Hz** |
| Depth MSE | 6.13e-02 | 0.897 崩溃 | 1.13 仍崩溃 | 1.05 仍崩溃 |

结论：
- **跟踪 MSE 全面突破**：roll/pitch/yaw 三轴 MSE 全部进入 e-3 级（roll 3.95e-3、pitch 3.65e-3、yaw 2.16e-3），分别为基线的 1/14、1/15、1/25。这是「B1 不动 + D1 弱惩罚」的最佳组合点。
- **ripple_std 略升、主频回到 2-3Hz**：相比 D1 full，ripple_std 从 0.04→0.05-0.06，fft 主频从 1.7-1.9Hz 回到 2.3-3.0Hz。说明 D1 强度下降时纹波抑制效应也线性回退，但 MSE 收益远超纹波代价 —— **当前是更好的折中点**。
- **Yaw 自然达成最佳值**：B1 回退后 yaw 表现仍最优（MSE 2.16e-3、ripple_std 0.04），证明 B1 增益调整对 yaw 帮助有限，**真正发挥作用的是 D1 阻尼项**。
- **Depth 崩溃稳定保持**：3 个 stage1 run 都是 0.9-1.1，再次确认与 B/D 调整无关，是单 seed stage1 收敛方差问题，需另立专项。
- 决策：当前 (B1 回退 + D1×1/4) 作为新 baseline，进入 A3（D 项改真实角速度 + 观测加 ang_vel），看能否进一步压低 ripple_std。

### 7.4 A3：D 项改真实角速度 + 观测加 ang_vel（已验证 → 三方面史上最佳）

**动机**：原 D 项是 `actions_diff`（动作一阶差分），高频抖动主因之一是 actions 本身就含 RL 噪声；改用 `-ω_b · dt_ctrl`（真实 body 角速度乘控制周期）后，D 项语义恢复"对真实姿态变化的反作用"，理论上对噪声免疫。

**实施**（4 处代码改动）：
- 基类 `EasyUUVEnvCfg` [L160-L164](../../easyuuv_env.py#L160-L164) 新增两 cfg 开关 `d_use_ang_vel = False` / `obs_include_ang_vel = False`（默认关闭，4 维 baseline 不受影响）。
- `_compute_dynamics` [L1170-L1183](../../easyuuv_env.py#L1170-L1183) 当 `d_use_ang_vel=True` 时把 D 项替换为 `-self._robot.data.root_ang_vel_b * dt_ctrl`，depth 维仍用动作差分。
- `_get_observations` [L1106-L1112](../../easyuuv_env.py#L1106-L1112) 当 `obs_include_ang_vel=True` 时拼接 `root_ang_vel_b` (3 维)，让 policy 看到 D 项里的真实角速度。

### 7.5 A1：S 面 D 项 EMA 低通（已验证 → 回退）

**动机**：A3 把 ripple_std 压到 0.01 量级，但 fft 主频仍在 1.8-2.4Hz；理论上加一阶 EMA（tau=0.05s，截止 ~3.2Hz）可进一步抑制残余高频抖动。

**实施**（保留代码，仅 cfg 关闭，便于后续复用）：
- 基类 `EasyUUVEnvCfg` [L165-L167](../../easyuuv_env.py#L165-L167) 新增 `d_filter_tau`（默认 0.0）。
- `__init__` [L302-L303](../../easyuuv_env.py#L302-L303) 增 `_actions_d_filt` 状态 buffer。
- `_compute_dynamics` [L1184-L1191](../../easyuuv_env.py#L1184-L1191) 在 `actions_d` 进 `_pid_control` 前一阶 EMA：`alpha = dt_ctrl / (tau + dt_ctrl)`。
- `_reset_idx` [L906-L908](../../easyuuv_env.py#L906-L908) reset 时清零滤波状态。
- `EasyUUVParametricEnvCfg` 启用 `d_filter_tau = 0.05`，跑 1200 iter（Checkpoint：`logs/.../2026-06-08_13-23-01_stage1/model_1199.pt`）。

**对比（A3 vs A1）**：

| 指标 | A3 | A1 (tau=0.05) | Δ |
|---|---|---|---|
| Roll MSE | 2.25e-3 | 2.80e-3 | **+24%** |
| Pitch MSE | 1.65e-3 | 2.07e-3 | **+25%** |
| Yaw MSE | 7.85e-4 | 9.10e-4 | **+16%** |
| Depth MSE | 9.11e-2 | 9.26e-2 | +2% |
| roll ripple_std | 0.0115 | 0.0106 | −8% |
| pitch ripple_std | 0.0195 | 0.0197 | ~0 |
| yaw ripple_std | 0.0088 | 0.0094 | +7% |

**诊断**：A1 在 A3 之上是**双重低通**——
- A3 的 D 项已经是 `-ω_b · dt_ctrl`，`dt_ctrl=1/60s` 本身已把高频角速度幅度按 1/60 缩放，等价于一个隐式低通。
- 再叠 tau=0.05s 的 EMA 引入额外相位滞后，D 项响应性下降 → P 项主导 → 跟踪 MSE 退步 ~20%；但残余高频本来就被 A3 压住了，EMA 的边际抑制收益接近 0 → ripple_std 几乎不动。

**结论**：A1 在 A3 已开启的前提下**没有性价比**，回退 cfg 到 `d_filter_tau = 0.0`。代码实现保留，留给：
- 未来若 D 项改回 `actions_diff`（即不开 A3）的退路场景；
- 实物部署时 IMU 噪声更大、`-ω_b` 自身需要平滑的情况。

**最终 stage1 决策点 = A3**（D 项 ω + 观测 ang_vel + D1×1/4 阻尼奖励 + Yaw 1.0/0.13 原始增益）。

#### 7.7.3 异常点识别

**asymmetric（com_to_cob xy=0.05/0.05 m）+ STDW on：从历史"−44%"翻转为"+158%"**。三档 wave 下劣化幅度均 ≥ 156%（calm +156%、medium +161%、storm +158%）。直观解释：
- 旧 ckpt 在 asymmetric 上 fmse_off 已偏高（基线差），STDW 修偏后变好；
- A3 stage2 ckpt 在 asymmetric 上 fmse_off 已经达到 0.226（与 base 持平），意味着 A3 的 ω-based D 项 + obs 中的 ang_vel 已经把不对称体现出的姿态扰动学到位；
- 此时 STDW drift target +0.05 m 是**纯外部扰动**，不再是修偏，直接劣化。

**heavy_moderate**（旧 ckpt 上 +75% 反向劣化的"异常点"）在新 baseline 下退化为温和的 +5.3%——A3 stage2 已把这个原本最棘手的 emb 拉回正常区。

#### 7.7.4 Tune 维度（mean fmse_off / fmse_on）

| Tune | fmse_off | fmse_on |
|---|---:|---:|
| identity（旁路 4 机制） | 0.2311 | 0.2641 |

#### 7.7.5 关键现状判断

1. **STDW 在 base/long_body 是无差别加速器**（−66.5% / 任何 wave），可直接默认启用。
2. **STDW 在 asymmetric 是污染源**（+158%）——必须做 7.7.6 §1 提到的部署期 gating。
3. **A3 stage2 的"环境鲁棒边界"** ≈ 体型 + 海况，但**不包含质心-浮心 xy 偏移**。
4. **Tune full 起作用了**——说明 8D meta-control head 在 stage2 后真正进入工作区，可以把后续优化聚焦在 a_gain 调节策略上。

#### 7.7.6 提升方向（按优先级）

**P1. 部署期 STDW gating（必做，研究门槛低）**
- 短跑 200 step STDW off 估出 fmse_short；若 < 0.10 m² 阈值则关闭 STDW，否则启用；
- 这能避免 asymmetric 上 +158% 的劣化，把全局 fmse 从 0.252 降到 ≤ 0.142。

**P2. asymmetric 鲁棒化专项（直击异常点）**
- 候选：训练时 domain randomize com_to_cob xy ∈ ±0.05 m，让 stage2 增益头学到"已处于不对称配重"的判别；
- 或：在 STDW 慢环里加入 com_to_cob 估计支路，把 drift target 改为"反方向修正"。

**P3. Tune full 的扩展验证**
- 当前 full 比 identity 多 8.8% 改善，但样本只有 24 配对单 seed；
- 跑 3 seed × 24 cell（off/on 配对仅取 full）= 72 trial，~30 min，给 full 模式一个置信区间。

**P4. 多 seed 全矩阵**（论文级 sanity check）
- 当前单 seed，无置信区间。3 seed × 48 cell = 144 trial ~70 min，完成后才能写"显著改善"。

**P5. yaw stage2 副作用专项**（来自 7.6 节遗留）
- yaw MSE +55% 是已知 stage2 副作用，与本节"asymmetric 翻转"可能同源（都是 stage2 的姿态-深度耦合补偿在某些 emb 上变成扰动）。
